# 巡逻

每次执行检查以下内容，有异常时通知用户，无异常时静默。

## 检查清单

### 1. 引擎健康

- 检查 `logs/` 目录最新日志文件的时间戳，超过 10 分钟没更新 → 引擎可能挂了
- 检查 `records/ledger.json` 最后修改时间，超过 5 分钟没更新 → 引擎可能挂了
- 检查进程列表有无 `hft-engine` / `data-engine` 在运行

### 2. 数据引擎健康

- 检查 `data/candles/` 各币种的 1m.parquet 最后修改时间
- 超过 2 小时没更新 → 数据引擎可能断了

### 3. 全局风控

- 读取 `records/ledger.json`
- 计算总虚拟权益 vs 初始资本（$222）
- 回撤 >= 7% → 预警（接近 10% 红线）
- 回撤 >= 10% → 紧急告警

### 4. 策略健康

- 遍历 ledger 中每个策略的虚拟权益和回撤
- 回撤 >= 25% 配额 → 标记为红灯
- 连续无交易 > 48 小时 → 标记为异常

### 5. issues/ 未处理

- 检查 `issues/` 目录有无未解决的 issue

## 输出

**无异常**：输出一行 "巡逻完成，一切正常 ✓" 然后结束

**有异常**：列出所有异常项，按严重程度排序：
```
⚠️ 巡逻发现异常：

🔴 全局回撤 8.5%，接近 10% 红线
🔴 bard-rsi-5m 回撤 28%，已超 25% 红灯线
⚠️ 数据引擎 LYNUSDT 2 小时未更新
⚠️ issues/ 有 1 个未处理问题

建议操作：
1. 检查 bard-rsi-5m 是否需要暂停
2. 检查数据引擎日志 logs/data-engine.log
```

## 定时使用

```
/loop 4h /patrol
```
