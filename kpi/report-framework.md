# ClawChat 报告体系

## 报告类型

### 1. P&L 日报（每日结算）
**频率**：每日收盘
**内容**：
- 今日已实现盈亏（已平仓交易的利润）
- 今日未实现盈亏（持仓浮盈/浮亏）
- 净值变化（昨日净值 → 今日净值）
- 收益率（日/周/月/累计）
**命令**：`make pnl`

### 2. 策略归因报告（每日）
**频率**：每日
**内容**：
- 各策略利润排名
- 各策略交易笔数、胜率、平均每笔利润
- 策略贡献占比（谁在赚钱）
- 策略状态（dryrun/live/stopped）
- 建议：加码/保持/砍掉
**命令**：`make status`

### 3. 风控报告（实时）
**频率**：每分钟（心跳）
**内容**：
- 总敞口（实盘资金量）
- 总浮亏 vs 红线
- 各策略距止损线的距离
- 价格偏离网格区间的程度
- 异常预警
**命令**：`make check`

### 4. 持仓报告（每日）
**频率**：每日
**内容**：
- 各币种持仓量和市值
- 持仓集中度（单币种占比）
- 子账户资金分布
**命令**：`make account` + `make projects`

### 5. 运营/系统报告（每日）
**频率**：每日
**内容**：
- 策略进程存活率
- API 调用成功率
- 邮件发送成功率
- 系统异常日志
**命令**：`make status`

### 6. 迭代报告（每次提交）
**频率**：每次 git commit
**内容**：
- 代码变更摘要
- 新增/修改的策略和技能
- 待办事项
**自动**：commit 时自动发送

### 7. 市场扫描报告（每小时）
**频率**：每小时
**内容**：
- 高波动币种排名
- 当前策略覆盖的币种表现
- 新机会推荐
**命令**：`make scan`

## 报告发送规则

| 报告 | 频率 | 发邮件 | 存本地 |
|------|------|--------|--------|
| P&L 日报 | 每日 20:00 | ✅ | ✅ |
| 策略归因 | 每日 20:00 | ✅ | ✅ |
| 风控报告 | 每分钟 | 异常时发 | ✅ |
| 持仓报告 | 每日 20:00 | ✅ | ✅ |
| 运营报告 | 每 30 分钟 | ✅ | ✅ |
| 迭代报告 | 每次 commit | ✅ | ✅ |
| 市场扫描 | 每小时 | ❌ | ✅ |

## 日报合并

每日 20:00 发送一封**综合日报**，合并 P&L + 策略归因 + 持仓 + 风控，一封邮件看全貌。
