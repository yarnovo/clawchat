# Agent 异常处理

## 1. 异常类型总览

| 异常类型 | 用户感知 | 自动处理 | 人工介入时机 |
|------|------|------|------|
| 回复超时 | 思考中 → 无法响应 | 检查容器状态，自动重启 | 容器假死时 |
| API Key 不足 | Agent 发通知告知 | 标记状态，暂停接单 | owner 充值后恢复 |
| 容器崩溃 | Agent 显示离线 | unless-stopped 自动重启 | 连续失败 3 次后 |
| Relay 断连 | Agent 显示离线 | 指数退避重连 | 10 分钟未连上 |
| 回复内容异常 | 友好提示重试 | 静默重试一次 | 无需 |
| Relay 服务挂了 | 网络不稳定提示 | 自动重启 + HTTP 降级 | 重启失败时 |

## 2. 回复超时处理

```
用户发消息
  → 超过 30 秒：显示「Agent 正在思考中，请稍候」
  → 超过 120 秒：显示「Agent 暂时无法响应，请稍后重试」
  → 后台自动检查容器状态 → 容器假死 → 自动重启
```

## 3. API Key 余额不足

```
Agent 调用 API 返回 402
  → Agent 发消息给用户：「我的 API Key 余额不足」
  → 推送通知给 owner
  → 状态标记为 api_key_exhausted
  → owner 充值后手动重新激活
```

## 4. 容器崩溃处理

```
容器意外退出
  → Relay 检测到 WebSocket 断开 → Agent 状态变为 offline
  → Docker restart policy 自动重启
  → 成功：重连 Relay → 恢复 online
  → 连续失败 3 次：标记 error → 通知 owner
  → 用户侧：「Agent 暂时离线，消息将在恢复后送达」
```

## 5. Relay 断连重连策略（指数退避）

| 重连次数 | 等待时间 |
|------|------|
| 第 1 次 | 立即重连 |
| 第 2 次 | 5 秒后 |
| 第 3 次 | 15 秒后 |
| 第 4 次 | 60 秒后 |
| 超过 10 分钟 | 上报 agent-manager |

## 6. Relay 服务宕机（最严重）

```
Relay 服务挂了 → 所有 Agent 同时离线
短期降级：消息走 HTTP 接口
监控告警：立刻发邮件，Docker 自动重启 Relay
```

## 7. 回复内容异常

| 异常内容 | 处理方式 |
|------|------|
| 空内容 | 静默重试一次，还是空则提示「请重新发送」 |
| API 报错 JSON | 过滤掉，显示友好提示 |
| 乱码 | 过滤处理 |
