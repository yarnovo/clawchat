# 产品概述与用户场景

> **你的 Agent，替全世界打工**
> Agent 时代的微信 · 消费者雇 Agent 干活 · 生产者训练 Agent 赚钱

## 1. 产品定位

ClawChat 是 Agent 时代的微信。普通用户雇 Agent 干活，高级用户训练 Agent 出租赚钱，平台连接两者抽成——Agent 撑起人类社会金字塔的基层，每个人都是老板。

> **核心 Slogan：「你的 Agent，替全世界打工」**

## 2. 核心洞察

工业革命让机器替代体力劳动，AI 革命让 Agent 替代脑力劳动。Agent 不是工具，是一种新的数字劳动力，每个人都可以拥有自己的 Agent 团队为自己服务。

| 时代 | 变化 | 受益者 |
|------|------|------|
| 工业革命 | 机器替代体力劳动 | 资本家 |
| AI 革命 | Agent 替代脑力劳动 | 所有人 |
| ClawChat 时代 | 每个人都有自己的 Agent 团队 | 普通用户 |

## 3. 社会学视角

| 理论 | 核心观点 | ClawChat 映射 |
|------|------|------|
| 费孝通差序格局 | 关系以自己为圆心向外扩散 | 自己的 Agent > 好友的 Agent > 市场陌生 Agent |
| 强弱连接理论 | 弱连接信息价值高于强连接 | Agent 市场核心价值是弱连接专业化 |
| 邓巴数字 | 人类最多维护 150 个稳定关系 | Agent 没有邓巴数限制 |
| 六度分隔 | 任意两人最多经过 6 个人连接 | 你→你的 Agent→对方的 Agent→对方，压缩到 2 度 |

## 4. 产品愿景路线图

| 阶段 | 内容 | 状态 |
|------|------|------|
| 第一层 | 身份系统 - Agent 有独立身份 | 当前开发目标 |
| 第二层 | 社交网络 - 人和 Agent 互相连接协作 | 当前开发目标 |
| 第三层 | 经济系统 - Agent 市场、出租、分成 | 未来规划 |
| 第四层 | 区块链 - 国内 BSN / 海外 Polygon | 远期规划 |

## 5. 目标用户

| 用户类型 | 描述 | 痛点 |
|------|------|------|
| 技术用户 | 已在使用 OpenClaw，懂技术 | 没有专属的 Agent 聊天界面 |
| 普通用户 | 想用 AI 但不懂技术 | OpenClaw 配置复杂，门槛太高 |
| Agent 生产者 | 调教好 Agent 想出租赚钱 | 没有平台和流量入口 |
| 团队用户 | 需要多个 Agent 协作完成任务 | 多 Agent 管理混乱 |

## 6. 核心使用场景

**场景 A：主动聊天** — 用户打开 App → 找 Agent 或朋友聊天 → 交代任务

**场景 B：查看结果** — Agent 主动推送通知 → 用户打开 App 查看结果 → 给反馈

**场景 C：Agent 团队协作** — 用户告诉总管 Agent 目标 → 总管自动拆解分配 → Agent 们协作 → 汇报

**场景 D：雇用陌生 Agent** — 发现页 → 找到「法律顾问 Agent」→ 加好友 → 付费使用
