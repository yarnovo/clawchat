# 冷启动与开发规划

## 1. 冷启动策略

### 第一步：进入 OpenClaw 社区
- 在 OpenClaw Discord 发布 ClawChat
- 在 GitHub 开源前端代码
- 让技术用户先用起来

### 第二步：内容传播
- 技术用户天然爱分享
- 「我用 ClawChat 搭建了自己的 Agent 团队」
- Twitter/X、Reddit、V2EX 传播

### 第三步：破圈到普通用户
- 技术用户帮家人朋友创建 Agent
- 口碑传播到非技术圈

## 2. 渐进式商业披露策略

**阶段一（对外定位：OpenClaw 专属聊天 App）**
- 核心卖点：为 OpenClaw 设计的专属聊天界面
- 开放功能：注册登录、创建 Agent、私聊、好友、群聊
- 隐藏功能：市场、付费、分成（代码已实现，通过 Feature Flag 控制）

**阶段二（社交 + 留存）**
- 开放：Agent 层级关系、子 Agent、离线推送
- 目标：100 活跃用户

**阶段三（商业化）**
- 开放：订阅系统、Agent 市场、计费、分成结算
- 目标：月收入覆盖服务器成本

**阶段四（规模化）**
- 开放：评价系统、推荐算法、Web 版、出海

## 3. 功能优先级

| 优先级 | 模块 |
|------|------|
| P0 | Docker 环境、数据库 Schema、账号系统、Relay、私聊、OpenClaw 插件、Agent 容器 |
| P1 | Agent 创建流程、好友系统、群聊、文件上传、离线推送 |
| P2 | 付费系统、BSN 上链 |
