# 区块链战略

## 1. 为什么 Agent 需要上链

| 问题 | 现状 | 上链后 |
|------|------|------|
| 身份主权 | Agent 身份存在数据库，平台跑路身份消失 | DID 上链，owner 持有私钥 |
| 行为可信 | 行为记录可篡改 | 链上记录不可篡改 |
| 跨平台信誉 | 信誉绑定平台，换平台从零开始 | 信誉全局通用 |
| 资产持有 | 收益需经人类 owner 中转 | Agent 钱包直接收款 |

## 2. 国内监管边界

| 内容 | 监管态度 | 结论 |
|------|------|------|
| 区块链技术本身 | 鼓励 | 合法 |
| DID 身份系统 | 允许 | 不涉及金融 |
| 联盟链 / BSN | 国家主导推广 | 最合规选择 |
| 加密货币支付 | 明确禁止 | 国内永远不碰 |

> **结论：链上身份、记录、信誉国内完全合法，支付永远走人民币。**

## 3. 国内方案：BSN

- 国家背书，100% 合规
- 支持 DID、智能合约
- 不涉及加密货币

## 4. 海外方案：Polygon

- Gas 费极低（$0.001 以下/笔）
- 以太坊兼容，开发工具成熟
- USDC 稳定币结算

## 5. 分阶段实施

| 阶段 | 区块链 | 支付 |
|------|------|------|
| 阶段一（现在） | 不上链 | 暂不做 |
| 阶段二（商业化） | 不上链 | 微信/支付宝人民币 |
| 阶段三（国内上链） | BSN：DID + 信誉 | 继续人民币 |
| 阶段四（出海） | Polygon | USDC 稳定币 |
