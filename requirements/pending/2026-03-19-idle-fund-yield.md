# 闲置资金理财

**优先级**: P2
**来源**: 调研分析（收益优化）

## 问题

当前账户 reserve 资金（~$113，占总资金 ~50%）完全闲置：
- reserve 在 Binance 账户中不产生任何收益
- 策略配额仅使用了 $203/$293（虚拟），实际可用余额 $218
- 闲置资金的机会成本：按 Binance Earn 活期 5-8% 年化计算，$113 每年损失 $5.6-$9.0

虽然金额在当前阶段不大，但随着资金增长，reserve 比例会持续存在（风控要求保留一定比例），闲置成本会放大。

## 需求

### 1. Binance Earn 活期对接

- 接入 Binance Simple Earn API（活期申购/赎回）
- reserve 资金自动申购 USDT 活期产品
- 需要下单前自动赎回对应金额（确保流动性）

### 2. 资金分配规则

- reserve 的 80% 可用于活期理财（剩余 20% 保持即时可用）
- 赎回延迟 < 1 分钟（活期产品特性）
- 策略开仓前自动检查并赎回所需金额

### 3. 收益追踪

- 理财收益记入 ledger（独立类目：earn_income）
- 日报中展示理财收益
- 写入 `records/earn.jsonl`

### 4. 安全约束

- 仅使用活期产品（随时可赎回），不使用定期
- 仅申购 USDT 稳定币产品（无价格波动风险）
- 赎回失败时不阻塞交易（降级为不申购）

## 涉及文件

- `shared/src/exchange.rs` — Earn API 接口（申购/赎回/查询）
- `engine/src/ledger.rs` — 理财收益记账
- `engine/src/order_router.rs` — 开仓前赎回检查
- `ops/src/` — earn 查询命令
