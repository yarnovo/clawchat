# WS 连接每 75 秒超时

**发现来源**: dry-run 测试
**严重程度**: 中（有自动重连，不会崩溃，但每 75 秒断一次会丢行情）

## 现象
所有 Binance WS 连接（aggTrade/markPrice/userDataStream）每 ~75 秒统一超时断开。

## 已排除
- combined stream 合并（从 11 条降到 3 条，问题依旧）
- 主动 ping 保活（每 20 秒发 ping，问题依旧）

## 待排查
- 关闭 VPN/代理测试（最可能的根因）
- macOS TCP keepalive 设置调整
- 换网络环境测试

## 已解决
开启代理的 tun 模式（虚拟网卡）后问题消失。根因是 HTTP 代理模式下 WS 长连接被代理 75 秒超时断开。

**解决方案**: 代理使用 tun 模式（虚拟网卡），不要用 HTTP 代理模式。
