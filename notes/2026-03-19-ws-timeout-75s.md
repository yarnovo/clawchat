# Binance WS 75 秒超时问题

## 现象
交易引擎 WS 连接成功后约 75 秒统一超时断开：`IO error: Operation timed out (os error 60)`

## 根因
macOS TCP keepintvl 默认 75 秒。网络中间设备（代理/VPN）在连接空闲 60-75 秒后静默丢弃，TCP 层以 75 秒间隔报告超时。

## 修复
在所有 WS 循环中每 20 秒主动发 WebSocket Ping 帧保活，防止连接被判定为空闲。

## 相关文件
- engine/src/gateway.rs — combined_feed_loop
- engine/src/main.rs — start_user_data_stream, start_combined_mark_price_feed

## 最终解决
开启代理 tun 模式（虚拟网卡）后问题消失。HTTP 代理模式会在 75 秒后断开 WS 长连接。

## 经验
- 在中国连 Binance 必须走代理
- HTTP 代理模式不支持 WS 长连接（75 秒超时）
- tun 模式（虚拟网卡）让所有流量走代理，WS 长连接正常
- 部署到海外 VPS 是长期最优方案

## 参考
- Binance WS 服务端 ping 间隔 3 分钟，10 分钟无 pong 断开
- macOS `net.inet.tcp.keepintvl` = 75000ms
